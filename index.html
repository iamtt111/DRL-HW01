V(s)：狀態價值函數，表示從狀態 s 開始的期望累積獎勵

π(s)：策略，表示在狀態 s 應採取的動作 (↑↓←→)

演算法說明

Policy Iteration:
1. 初始化隨機策略 π
2. Policy Evaluation：評估當前策略
V(s) ← Σa π(a|s) Σs',r p(s',r|s,a)[r + γV(s')]
3. Policy Improvement：改進策略
π(s) ← argmaxa Σs',r p(s',r|s,a)[r + γV(s')]
4. 重複 2-3 直到 π 收斂
Value Iteration:
1. 初始化 V(s) = 0, ∀s
2. 對每個狀態取最佳價值動作，Bellman Update
V(s) ← maxa Σs',r p(s',r|s,a)[r + γV(s')]
3. 重複直到 V(s) 收斂 (Δ < θ)
4. 從 V(s) 提取策略
π(s) ← argmaxa Σs',r p(s',r|s,a)[r + γV(s')]

參數： γ = 0.9 (折扣因子) | θ = 0.01 (收斂閾值)

符號： π(a|s) = 策略機率 | p(s',r|s,a) = 狀態轉移機率 | 本環境為確定性 (p=1)
